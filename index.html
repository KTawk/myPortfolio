How I Work

I start every project by clarifying user needs and goals. I talk with stakeholders or end users, jot down their main points, and turn those insights into clear requirements. Then I sketch quick wireframes, basic layouts that map out buttons, menus, and content areas without any styling, so everyone agrees on the structure before we write any code.

Next, I build the functionality step by step. I break work into small tasks, write clean maintainable code with consistent naming, and add automated tests to catch regressions early. I use Git and GitHub to collaborate, open issues to track progress, and hold short check ins with stand ups or async updates to stay aligned on scope and blockers.

Finally, I deploy and learn. I share early beta builds, monitor real user feedback and metrics, and then refine the experience in quick cycles. I am always learning—I am currently taking a UI design course where I study core design elements. I also read articles to keep my techniques fresh. Check it out!
